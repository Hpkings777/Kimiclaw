# 基础使用建议

## 推荐的使用方式
- 安装 app 后，开启“完全运行后台行为”权限，并将 app 放在前台使用
- 关闭系统自动升级功能，避免自动升级导致手机系统重启

## 如何使用操作 app 功能
- 开启无障碍权限后，保持屏幕处于解锁状态

## 备份与恢复
- 重要配置建议通过设置中的“备份与恢复”功能加密备份到手机存储中，路径为 Documents/KimiClaw 目录
- 恢复备份功能，备份文件包含 agent 配置、技能配置和一些日志，恢复时会覆盖当前配置，请谨慎使用

# Gateway 相关问题

## agent 不回应聊天
- 检查手机网络连通性
- 检查 agent 是否在运行，或重启 agent

## Gateway 启动失败
- 先尝试仪表盘"重启"按钮；不行就进设置点"一键修复"
- 默认端口 18789，如被占用会导致启动失败
- 最后的修复手段是清除 app 数据，路径为设置 -> 存储 -> 清除数据，注意这会清除 agent 配置、技能配置和一些日志，所以要提前使用设置中的备份功能

## 华为系统杀后台
- 华为系统可能会杀掉后台进程，导致 agent 无法正常工作。建议在华为手机上使用时，进入系统设置 -> 应用 -> KimiClaw -> 耗电详情 -> 启动管理 中，关闭“自动管理”，并开启“允许后台活动”、“允许关联启动”和“允许后台活动”。

## 小米系统
- 小米系统会杀后台进程，所以需要赋予 KimiClaw 自启动和后台运行权限。进入系统设置 -> 应用 -> 管理应用 -> KimiClaw -> 权限管理，开启“自启动”和“后台活动”。
